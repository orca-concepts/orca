PRIVACY POLICY

Orca Concepts LLC, a Wisconsin limited liability company (orcaconcepts.org) (“Company” or “Orca”) welcomes you to its online service (the "Service"). By using its Service, you (“You” or “User”) are agreeing to the following, whether or not you are a registered customer or User of Orca (orcaoncepts.org). This Privacy Policy (“Privacy Policy”) may be updated from time to time without notice to you. Capitalized terms or other terms not herein defined are defined in the Subscriber Agreement and Terms of Use (“Agreement”) (orcaconcepts.org/terms). This Privacy Policy applies to personal data collected in connection with the Company Website(s) and the Website’s processing activities, as reflected in the Agreement.

  1. DATA WE COLLECT.

    1. Account data (collected at registration):

      1. Username (chosen by User)

      2. Phone number (used for OTP verification during registration and password reset only)

      3. Email address (used for legal notifications, copyright violation notices, and other policy updates); it is not publicly visible

      4. Password (stored as a one-way hash; used for login)

    2. Optional Account Data (User-initiated)

      1. ORCID ID (verified via OAuth with orcid.org; stored if the User chooses to link their ORCID account)

    3. How Phone Numbers are Stored

      1. Phone numbers are hashed using bcryptjs (one-way hash; cannot be reversed to retrieve the original number)

      2. A separate HMAC-SHA256 deterministic hash is stored for O(1) lookup during password reset (keyed by a server-side secret).

      3. The original plain-text phone number is NEVER stored in the Company’s database.

      4. Phone numbers are sent to Twilio (third-party service) for OTP delivery during registration and password reset. Twilio's privacy policy governs their handling of phone numbers.

    4. How Passwords are Stored

      1. Passwords are hashed using bcryptjs (10 salt rounds). The original plain-text password is NEVER stored.

  2. DATA WE DO NOT COLLECT

    1. Location data

    2. IP addresses (beyond what is inherent in server logs)

    3. Tracking cookies or analytics data

    4. Browser fingerprints

    5. Third-party advertising or analytics identifiers

    6. Data from anyone under age 18

  3. USER ACTIVITY DATA

User activity on the Website varies in its visibility:

  1. Attributed by username: Annotations on Content or Submissions, web links added to Concepts, document uploads (uploader name visible), Corpus ownership, Superconcept ownership, page comments, moderation comments

  2. Visible but not attributed: Concepts and edges created in the graph hierarchy, votes (graph votes, swap votes, link votes, tunnel votes). These are stored with the User's ID internally but the username is not displayed in the user interface (“UI”).

  3. Attributed within limited contexts: Corpus membership (usernames visible to other Users who are members of the same Corpus), document co-authorship (visible to other co-authors).

Users with verified ORCID IDs will have a small ORCID icon displayed alongside their username wherever it appears in authorship or membership contexts.

  1. DATA WE SHARE AND DO NOT SHARE

    1. We do not sell, rent, or share User data with third parties for marketing or advertising purposes.

    2. Phone numbers are transmitted to Twilio solely for the purpose of sending OTP verification codes during registration and password reset. No other User data is shared with Twilio.

    3. ORCID verification: During the optional ORCID linking process, the User is redirected to orcid.org to authenticate. Orca receives only the verified ORCID ID back from ORCID — no access tokens are stored. This is subject to ORCID's privacy policy (https://info.orcid.org/privacy-policy/ ).

    4. Planned: Document files will be stored using Cloudflare R2 (cloud object storage). Cloudflare will store the uploaded file data but will not have access to user identity information. This is subject to Cloudflare's privacy policy (https://www.cloudflare.com/privacypolicy/ ).

    5. As an open-source platform, the source code is publicly available, but the DATABASE (containing User data and Content) is not.

  2. COOKIES AND LOCAL STORAGE

    1. Orca uses a JWT (JSON Web Token) stored in the browser for authentication session management. Local storage is used for UI preferences (e.g., attribute filter defaults on the root page).

    2. No third-party cookies, tracking pixels, or analytics scripts are used.

  3. DATA RETENTION

    1. Account data (username, hashed phone number, email, hashed password, ORCID iD) is retained until the User deletes their account.

    2. User-generated Content follows the four-tier policy described in the Agreement:

      1. Concepts, edges, annotations, web links, tunnel links, and Superconcepts are retained permanently (creator field cleared on account deletion where applicable).

      2. Votes, subscriptions, and tunnel votes are deleted when the account is deleted.

      3. Documents follow Corpus membership lifecycle; orphaned documents are auto-deleted (unless rescued by the uploader). Uploaders can also permanently delete their own document versions at any time.

    3. Dormant tables and retired features: Some database tables are retained per the platform's append-only philosophy even after the associated feature is retired. This data is inert and not actively processed.

  4. DATA SECURITY

    1. Phone numbers are hashed using industry-standard bcryptjs (10 salt rounds) and HMAC-SHA256. Passwords are hashed using bcryptjs (10 salt rounds) and validated for strength using the zxcvbn library.

    2. Authentication tokens (JWTs) can be invalidated globally via "Log out everywhere" feature.

    3. The Website(s) uses rate limiting on authentication endpoints (5 requests per IP per 15 minutes on OTP sending; 10 requests per IP per 15 minutes on login).

    4. ORCID integration uses server-to-server OAuth token exchange — ORCID access tokens are used once for verification and not stored.

    5. We take other industry standard security measures to protect data.

  5. USER RIGHTS

    1. Access: Users can view their contributions through the Website interface: voted Concepts are visible via graph votes, uploaded documents appear in "My Documents" sections within each Corpus, and annotations can be filtered by authorship using the Author filter on individual documents. Users can view their profile page showing their username, ORCID ID (if linked), and activity statistics. There is not currently a single unified view of all contributions.

    2. Deletion: Users can delete their account, which removes personal data and clears creator attribution on contributions (see Section 17 of the Agreement). Users can also permanently delete specific document versions they uploaded (see Section 6 of the Agreement).

    3. ORCID disconnection: Users can disconnect their ORCID ID at any time via their profile page.

    4. Our data export feature may be found on your profile page (click your username at the top and look under Privacy and Data). Please contact us at orcaconcepts@gmail.com if you believe your personal data needs correction or you need assistance with data export.

  6. THIRD PARTY SERVICES

    1. Twilio: Used for phone number OTP verification during registration and password reset. This is subject to Twilio's privacy policy (https://www.twilio.com/legal/privacy ).

    2. ORCID: Used for optional User (researcher) identity verification via OAuth. Only the verified ORCID ID is received and stored; no access tokens are retained. This is subject to ORCID's privacy policy (https://info.orcid.org/privacy-policy/ ).

    3. Cloudflare R2 (planned): Will be used for cloud storage of User-uploaded document files. This will be/ is subject to Cloudflare's privacy policy (https://www.cloudflare.com/privacypolicy/ ).

    4. Railway (planned): Will be used for application hosting (web server and database). This is subject to Railway's privacy policy (https://railway.com/legal/privacy ).
