  1. WHAT ORCA IS

    1. Orca is a collaborative web platform for building shared concept hierarchies (“Ontologies” or “Ontology”) and annotating research documents.

    2. Users create hierarchical graphs of concepts (e.g., ‘Methodology [value] > Reproducibility [value] > Pre-Registration [value]’), organize research documents into collections called Corpuses (as defined below), and annotate those documents by linking text passages to concepts in the graph.

    3. The Website platform is designed for productive contestation — Users can vote on concepts in the graphs, disagree, propose alternatives (swap votes), and flag content for community review. Quality emerges from community norms and transparent voting, not algorithmic curation.

    4. The Website platform uses pseudonymous identity; usernames are freely chosen and not verified against any real-world identity. All actions are tied to an account with a phone number, but Users are not required to use their real name. Usernames are visible in certain contexts (annotations, document uploads, Corpus/Superconcept ownership, web links, and page comments) but are not displayed on all contributions. For example, concept and edge creation in the graph hierarchy does not display who created them.

    5. Users can optionally verify and link their ORCID ID (a researcher identifier) via OAuth. Verified ORCID IDs appear as small icons next to usernames where authorship or membership is displayed.

    6. The Website platform code is open source under the GNU Affero General Public License v3 (AGPL-3.0). This means anyone can view, copy, and modify the source code, but if they run a modified version as a network service, they must make their modifications publicly available under the same license. The open source license applies to the SOFTWARE ONLY, not to User-generated Content or Submissions, which is governed by this Agreement.

  2. CORPUSES (DOCUMENT COLLECTIONS)

    1. Corpuses are named collections of documents, created by a single User owner, and are considered Content within the meaning of this Agreement (“Corpus” or “Corpuses”).

    2. The User Corpus owner and all Corpus members (invited via link or added by username/ORCID search) can add documents to the Corpus.

    3. The User Corpus owner can remove any document from the Corpus. User Corpus members can remove only documents they personally added.

    4. The User Corpus owner controls membership and Corpus settings. Corpus names are unique (case-insensitive).

    5. Corpus ownership can be transferred to another User.

    6. Users must transfer ownership of all Corpuses before deleting their account.

  3. USER-GENERATED CONTENT: 4 TIERS

Orca has an append-only architecture. Most Content cannot be deleted; it can only be hidden, anonymized, or (in the case of documents) permanently removed by the original User uploader (or the Company). The 4 Tiers of Content are as follows:

  1. TIER 1: CONCEPT HIERARCHY CONTRIBUTIONS (concepts, edges, annotations, web links, tunnel links)

    1. These are User community Content contributions that others build upon.

    2. Once created, they CANNOT be deleted or removed by the User.

    3. Tunnel links (cross-graph connections between concepts) are permanent and bidirectional. Creating a link in one direction automatically creates the reverse link as well.

    4. Annotations cannot be individually deleted by any User. However, if the document version to which an annotation is attached is deleted by its User uploader, all annotations on that version are destroyed as part of the cascade.

    5. If a User deletes their account, these contributions remain in the system and on the Website(s). For items that display usernames (annotations, web links), the username is replaced with a generic placeholder.

    6. The Company’s rationale is that other Users' work (votes, child concepts, annotations) depends on the structural integrity of these User contributions. Removing them would break other User’s work. Note that the reference here to child concepts has nothing to do with age or minors.

  2. TIER 2: VOTES (graph votes, swap votes, link votes, annotation votes, web link votes, flag votes, tunnel votes)

    1. Votes are changeable. Users can add and remove their votes freely.

    2. Graph votes cascade: removing a vote also removes votes on all descendant edges in that branch.

    3. Graph votes and swap votes are mutually exclusive per edge: voting removes any existing swap, and vice versa.

    4. Tunnel votes are directional: voting for a connection in one direction is independent from votes in the other direction.

  3. TIER 3: UPLOADED DOCUMENTS

    1. Users upload research documents (“Submission” or “Submissions”) (.txt, .md, .pdf, .docx) into Corpuses.

    2. Submission documents are immutable once uploaded; text cannot be edited. Updates are accomplished by uploading a new version (version chain).

    3. The original User uploader provides the Submission document and can assign co-authors via invite tokens or by searching usernames/ORCID IDs.

    4. Upload affirmation: Upon upload the User must affirm they have the right to upload the document. Specifically, the User must affirm and acknowledge: “I affirm and expressly acknowledge I have the legal right to share or otherwise distribute this document or content.”

    5. Document deletion: The original User uploader (or any User co-author) can permanently delete any specific document version or Submission they uploaded. Deletion cascades to all annotations, messages, favorites, citation links, and cache entries for that version. Other versions in the chain are unaffected; if a downstream version referenced the deleted one, it becomes a standalone document.

    6. Document external links: User authors can attach external source URLs (arXiv links, DOIs, journal URLs) to documents or Submissions. Multiple links per document or Submission are supported.

    7. Annotation citation links: Users can copy citation URLs for annotations and paste them into external research document Submissions. When those document Submissions are later uploaded to Orca, cited annotations are automatically detected and displayed as linked references.

    8. Document Submissions can also be removed from Corpuses by the User Corpus owner, or by the User Corpus member who originally added the document Submission. If a document is removed from all Corpuses, it is auto-deleted (orphan cleanup), unless uploaded by a User Corpus member, in which case it is left orphaned for the User uploader to rescue or dismiss.

  4. TIER 4: SUPERCONCEPTS (Curated Annotation Collections)

    1. Superconcepts are User-created Content collections of concepts-in-context (edges) from across the graph system, presenting a unified view of all annotations attached to those edges (“Superconcept” or “Superconcepts”).

    2. Only the User Superconcept owner can add or remove edges from the Superconcept.

    3. Superconcepts cannot be deleted, consistent with Orca's append-only philosophy. A Superconcept with zero subscribers still exists.

    4. Superconcept ownership can be transferred to another User via username/ORCID search.

    5. Other Users can subscribe to Superconcepts and cast votes on annotations within the Superconcept context (separate from User Corpus-level votes).

    6. Users must transfer ownership of all Superconcepts before deleting their account.
